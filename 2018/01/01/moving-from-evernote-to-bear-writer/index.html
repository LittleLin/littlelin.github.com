從 Evernote 搬家到 Bear

2018-01-01

總算是到了這一天。在作為 Evernote 長達 8 年的使用者後 (其中有 5 年是付費用戶)，在今天，我決定將自己的筆記軟體，正式切換為 Bear。

這並不是一個非常困難的決定，尤其是近年來 Evernote 的功能更新，實在沒有我想要的。做為付費用戶，我越來越覺得我每年的 TWD 1,350 (或我可以降階改使用 TWD 750 付費方案) 花得不是非常值得，尤其是我一直期盼對於 Markdown 編輯的完整支援，即使到現在，也還是沒有非常的流暢。對這議題有興趣的朋友，可以看看以下的討論串︰

  • Native Markdown Support
  • 从 Evernote 开始官方支持 Markdown 说起

當然我是可以退回入門版方案，單純使用 Evernote 做為我的筆記軟體。但經過這些年，我發現我對於筆記軟體的需求也有所變更，對於 Markdown 語法的支援就是其中一個，我整理了一下我自己對於筆記軟體的要求如下︰

  • 支援 Markdown 編輯語法︰這點非常重要，尤其是 Markdown 在我的文字編輯上，佔有非常重要的角色。工作上在公司 Git 上的文件，和目前 blog 寫作上，都是使用 Markdown 語法，我希望未來的筆記軟體也可以使用 Markdown 語法，這樣我就可以減少許多工作、筆記、Blog 寫作上，格式內容的不必要轉換。
  • 流暢的鍵盤熱鍵功能︰Evernote 一直以來的熱鍵體驗非常不直覺，我想可能是因為功能過多，導致熱鍵使用上非常複雜。我從來就沒有記住 Evernote 的熱鍵，因此希望新的筆記軟體在這方面可以好用一點。
  • 少一點「協作」功能︰最近不知道為什麼？一堆筆記軟體都標榜自己的協作功能非常強大，但身為一個邊緣人，我只是想要好好地寫筆記，和朋友共編文件的機會，並不是非常的多 (而且工作上在公司我都是使用 Google Doc)。我希望新的筆記軟體，可以少強調這一塊的特性，多投入資源在筆記的編輯體驗上 (這一點也是我最終想離開 Evernote 的原因)。
  • 基本的設備間同步功能︰這應該不難理解，現在 Mac + iPhone 是我工作上的主力，我也希望可以讓筆記可以在這兩個設備上互通。
  • 文件間互連功能︰另一個我覺得 Evernote 做得不是很好的地方。因為我自己在做筆記上，常常會在筆記中，參照另一篇筆記的內容。比方說我在擬去美西流程時，可能會將交通方式寫在筆記 A，住宿方式寫在文件 B，這時候如果有方便的文件參照功能，對於我在寫筆記上會非常有幫助。

和選擇 blog 平台一樣，我就開始依照上述需求，來選擇我的下一個筆記軟體，下面是我評估過的筆記 App︰

  • Quip: 原本是我第一優先的筆記 app，對於 Markdown 的支援也算不錯 (但仍然不是那麼好)，但我覺得它有幾個缺點︰
    • 太強調協作功能︰甚至整個 app 的動線上，考量協作流程還比筆記體驗要多。
    • 對於 Markdown 的支援不是非常完整︰或者應該說操作體驗和我預期的不一樣
  • Ulysses: 其實差一點就要選擇它了。對於 Markdown 的支援很不錯，熱鍵設計也非常順，但它有一個硬傷，就是如果要在筆記中貼入程式碼，它的語法 highlight 和顯示上做得很差。因為做為工程師，我時不時可能還是需要貼一些 code snippet 在我的筆記中，所以這個硬傷對我來說還是挺嚴重的，最後就還是放棄了它。
  • Microsoft OneNote: 我覺得它用起來，還是太以 Office 軟體的體驗為主，所以也不是我想要的筆記 app。
  • Google Keep: 功能太過簡單，不符合我對筆記的要求。
  • Dropbox Paper: 另一個我覺得過度強調協作，而沒有把筆記體驗本身做好的 app。
  • Simplenote: 界面簡潔，但是目前還不支援 Markdown 語法，因此也不考慮。

幾經選擇後，最終就決定使用 Bear 做為我下一個主力使用的筆記軟體。Bear 也是 2017 年 Apple 最佳設計 app 的得獎主之一，對於我來說，Bear 有以下好用的地方，當然這些優點也滿足了上述我對於筆記軟體的需求︰

  1. 對於 Markdown 語法編輯的良好支援
  2. 聚焦在「做筆記」的核心功能上︰如同我上面所說，協作只是多餘功能，讓用戶「好好寫筆記」才是一個筆記軟體應該要提供的核心功能。這方面是我最喜歡 Bear 的地方，看得出來整個團隊在打磨核心功能所花費的心力非常多，令我在寫筆記的體驗上非常舒服
  3. 整體簡潔乾淨︰畢竟是 2017 年最佳設計 app 的得獎主之一。
  4. 熱鍵好用 & 好記︰這點也是我非常欣賞 Bear 的地方，我基本上不太需要花太多心力，就可以把 Bear 的熱鍵記下並操作流暢。

當然 Bear 也不是毫無缺點，它最大的硬傷就是目前只支援 Mac/iPhone/iPad，並不支援 Android 平台，這部份在官方 Reddit 討論區上，也有用戶提出相關需求︰Will bear ever come to Android or alternative platforms?，但看來團隊因為成本考量，只打算先推出 web 版，暫時還沒有推出 Android app 的打算。

不過因為我目前工作上，80% 的時間都是使用 Mac + iPhone，所以雖然 Bear 還沒有 Android app 對於我使用 Pixel 2 會有一點困擾，但還不算非常大，所以最終就確定使用 Bear 做為我接下來的主力筆記軟體了。

新的一年，新的筆記軟體，希望使用 Bear 可以對我的生產力帶來新的提升囉，加油！
